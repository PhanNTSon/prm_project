# **III. Software Design Description**

## **1\. System Design**

### **1.1 System Architecture**

The overall architecture of the system is based on the Client-Server model.

{{DIAGRAM:system_architecture}}

- **Mobile Client (Flutter):** Responsible for displaying the user interface, managing local State, handling navigation (GoRouter), and managing network communication (Dio). Session login information (JWT Token) is stored securely via `shared_preferences`.
- **VNPay Sandbox:** When a wallet top-up is initiated, the Flutter App opens an `InAppWebView` pointing to the VNPay checkout URL generated by the Backend. The user processes the payment, and VNPay redirects (callback) to a URL containing the transaction parameters. The App intercepts this URL and reports the result to the user.
- **Backend & Database:** Acts as the Single Source of Truth, handling business logic, validating transactions, and persisting data on PostgreSQL.

### **1.2 Package Diagram**

The Flutter project adopts the **Feature-First** architecture. This method splits the project into independent modules, allowing developers to work in parallel without code conflicts.

{{DIAGRAM:package_diagram}}

Each sub-feature (e.g., `storefront`) is further divided into a 3-Layer Architecture model:

- `data/`: Contains models (DTO) and repositories communicating with the API.
- `providers/`: Contains state management logic (ChangeNotifier/Provider) for that specific feature.
- `views/`: Contains screens and specific UI components (Widgets) of that feature.

## **2\. Database Design**

_The database design is kept in sync with the current PostgreSQL database of the Spring Boot Backend._

- **Main Tables:** `User`, `Game`, `Transaction`, `TransactionDetail`, `Library`, `Cart`, `Review`.
- **Structure:** A User can own multiple Games in their Library (N-N relationship resolved via the Library table). Each addition to the cart creates a record in the Cart table. Successful payment deducts the User's `WalletBalance` and generates a `Transaction` along with a `TransactionDetail`.
- **User Table Details:**
  - `UserID` (bigint, PK)
  - `Email` (varchar, Unique)
  - `Username` (varchar, Unique)
  - `WalletBalance` (numeric)
- **Game Table Details:**
  - `GameID` (bigint, PK)
  - `Name` (varchar)
  - `Price` (numeric)
  - `IconUrl` (varchar)
